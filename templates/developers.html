Overview

The PropertyData API provides programmatic access to Australian property and address data. All endpoints return JSON and follow standard HTTP conventions.

Typical flow: address-suggest → geocode → (lookup, ai-lookup, nearest-transport, nearest-schools, da, title-search) in parallel.

Authentication

All requests must include your API key in the X-API-KEY header.

HTTP
GET /api/v1/property/geocode?gnaf_id=GAACT717940975
X-API-KEY: your_api_key_here
Python
import requests

headers = {"X-API-KEY": "your_api_key_here"}
r = requests.get(
    "https://api.propertydataservices.com.au/api/v1/property/geocode",
    params={"gnaf_id": "GAACT717940975"},
    headers=headers,
)
print(r.json())
cURL
curl -H "X-API-KEY: your_api_key_here" \
  "https://api.propertydataservices.com.au/api/v1/property/geocode?gnaf_id=GAACT717940975"
Keep your API key secret. Do not expose it in client-side JavaScript.

Base URL

https://api.propertydataservices.com.au/api/v1/property

All endpoint paths below are relative to this base URL.

Errors

StatusMeaning
400Bad request — missing or invalid parameter
401Unauthorised — invalid or missing API key
404Resource not found (e.g. unknown G-NAF ID)
422Validation error — check parameter types
502Upstream API error (Geoscape, Gemini, etc.)

Error responses include a detail field describing the problem:

JSON
{"detail": "G-NAF address not found: GAACT000000000"}
GET /address-suggest

Returns address autocomplete suggestions for a partial query string. Backed by the Geoscape Predictive API.

Query Parameters

ParamTypeRequiredDescription
qstringYesPartial address query (min 3 chars)

Response

JSON
[
  {
    "gnaf_id": "GAACT717940975",
    "display_name": "1 Main Street, Sydney NSW 2000"
  },
  ...
]

Example

cURL
curl -H "X-API-KEY: your_key" \
  "https://api.propertydataservices.com.au/api/v1/property/address-suggest?q=1+Martin+Pl"

Returns cadastre parcel data for a coordinate pair from the NSW Spatial Services. Returns null if no parcel is found or the address is outside NSW.

Typical Workflow

Most integrations follow this three-step pattern:

Python — full workflow
import requests

BASE = "https://api.propertydataservices.com.au/api/v1/property"
H = {"X-API-KEY": "your_api_key_here"}

# 1. Suggest
suggestions = requests.get(f"{BASE}/address-suggest", params={"q": "1 Martin Pl"}, headers=H).json()
gnaf_id = suggestions[0]["gnaf_id"]

# 2. Geocode
geo = requests.get(f"{BASE}/geocode", params={"gnaf_id": gnaf_id}, headers=H).json()
lat, lon, address = geo["lat"], geo["lon"], geo["display_name"]

# 3. Parallel lookups (shown sequentially for clarity)
property_data = requests.get(f"{BASE}/lookup",
    params={"lat": lat, "lon": lon, "address": address,
            "suburb": geo["suburb"], "postcode": geo["postcode"], "state": geo["state"]},
    headers=H).json()

transport = requests.get(f"{BASE}/nearest-transport",
    params={"lat": lat, "lon": lon, "address": address, "radius_m": 1000},
    headers=H).json()

print(property_data)
print(f"{len(transport['stops'])} transport stops found")
